# План реализации пользовательского флоу заказа

## 1. ЦЕЛЬ
При нажатии кнопки «Заказать» (Запросить расчет) на странице подробностей товара (`ProductDetails`), перед пользователем должно открываться модальное окно. В этом окне пользователь вводит дополнительные данные (контакты, адрес, количество, метод оплаты и комментарии). Заказ должен регистрироваться на клиенте, сохраняться в базе данных через `orders-service` (с привязкой пользователя по JWT), а затем корректно отображаться в разделе «Профиль» пользователя и в Панели администратора (`/admin`).

---

## 2. FRONTEND — UI

### 2.1 ProductDetails
- Кнопка "Заказать" теперь открывает модальное окно `OrderModal`, а не отправляет запрос сразу.
- Передача в модалку данных о товаре (name, price, ID).

### 2.2 OrderModal (НОВЫЙ КОМПОНЕНТ)
**Файл:** `frontend/src/components/OrderModal.jsx`
- **Атрибуты формы:**
  - `name` (Имя)
  - `phone` (Телефон/WhatsApp)
  - `address` (Адрес доставки)
  - `whatsapp` (Связаться по WhatsApp - чекбокс)
  - `quantity` (Количество, по умолчанию 1)
  - `paymentMethod` (Способ оплаты)
  - `comment` (Комментарий)
  - `selectedOptions` (Размер, цвет)
- **Отображение инфо о товаре:**
  - Наименование `productName`
  - Цена `price`
- **Действия:** "Отмена" и "Отправить заказ"

---

## 3. FRONTEND — ЛОГИКА

### 3.1 submit заказа
- Отправка `POST /api/orders` с правильным телом JSON.
- Использование интерфейса `api` (axios) для корректной подстановки `Authorization: Bearer <token>`.

### 3.2 обработка ошибок
- Отображение ошибок (валидация на клиенте и ошибки с бэкенда).

### 3.3 уведомление об успехе
- Уведомление (toast/alert) и закрытие модалки.

---

## 4. BACKEND (orders-service)

### 4.1 обновить модель Order
- Внести изменения в схему Prisma:
  - `address` (String)
  - `whatsapp` (Boolean)
  - `quantity` (Int)
  - `paymentMethod` (String)
  - `comment` (String?)
  - `selectedOptions` (Json?)
- Обновить миграции базы данных.

### 4.2 обновить POST /orders
- Обработка входящих полей по новой схеме БД при создании заказа.
- Брать `userId` безопасно из токена.

---

## 5. PROFILE PAGE

- На странице `GET /api/orders/my` доработать отображение UI-элементов заказа пользователя.
- **Отображать новые поля:**
  - `productName`
  - `status`
  - `quantity`
  - `address`
  - Выбранные опции `options`

---

## 6. ADMIN PANEL

- Улучшить список заказов (вывод расширенной информации).
- Детальное отображение данных клиента (адрес, телефон).
- Возможность смены статуса (сохраняется).

---

## 7. ИНТЕГРАЦИЯ

- Связь всего потока: **Product → Order → Profile → Admin**.

---

## 8. ТЕСТЫ

Пошаговый сценарий ручного тестирования:
1. Открыть любой товар в каталоге.
2. Нажать «Запросить расчет»/«Заказать».
3. В открывшейся модалке правильно заполнить все новые поля.
4. Отправить заказ, получить подтверждение успеха.
5. Проверить в `/profile` наличие заказа с правильно отображенным количеством и адресом.
6. Переключиться на Аккаунт Администратора, зайти в `/admin` и проверить появление нового заказа со всей информацией и сменить его статус.
