# Дипломдық жұмысқа қажетті суреттер, логикалық блок-схемалар және скриншоттар тізімі

Бұл файлда дипломның мәтініне (тарауларына) сүйене отырып, қандай суреттер мен схемаларды салу немесе түсіру керектігі нақты көрсетілген. **Әр тақырыпшаның соңында немесе мәтінінің ішінде (алдын ала «...Сурет 1-де көрсетілген» деп сілтеме жасай отырып) осы суреттер тұруы керек.**

---

## 1 Тарау. Пәндік аумақты талдау және есептің қойылымы
Бұл бөлімде жүйенің алгоритмдік және логикалық сызбалары (UML / DFD) болуы керек. Кез келген схеманы *draw.io* немесе *Microsoft Visio* арқылы сызуға болады.

- **1.3 Жүйеге қойылатын функционалдық және техникалық талаптар** бөлімінде:
  - **Сурет 1. Жүйенің Use-Case (Қолдану нұсқалары) диаграммасы.** 
    *(Бұл сызбада: Ортада шекара. Сол жақта «Тұтынушы» (Тауар іздеу, Себетке салу, Тапсырыс беру, Тіркелу қызметтері), оң жақта «Әкімші» (Тапсырысты басқару, Excel жүктеу, Аналитика қарау) үлгісінде сызылады).*

  - **Сурет 2. DFD мәліметтер ағынының диаграммасы.**
    *(Бұл сызбада: Клиенттен шыққан мәліметтердің (мысалы себетке салу), сервер арқылы және API Gateway арқылы өтіп, базаға (PostgreSQL) қалай жазылатындығы көрсетілген стрелкалы блок-схема).*

---

## 2 Тарау. Қосымшаның интерфейсі мен функционалдығын жобалау
Бұл бөлімде бағдарламаның бэкенд немесе дерекқор архитектурасының техникалық сызбалары орналасады (Бұларды да draw.io арқылы сызасыз).

- **2.1 Жүйенің архитектурасын таңдау және негіздеу** бөлімінде:
  - **Сурет 3. Жобаның микросервистік архитектура схемасы.** 
    *(Сызбада: Ең жоғарыда Фронтенд (React) -> Төменде API Gateway (8080) -> Одан екі сызық айырылады: Products Service (3001) және Orders Service (3002). Және бұлардың астында өздерінің тәуелсіз дерекқорлары).*

- **2.2 Жүйенің құрылымын және деректер қорының моделін жобалау** бөлімінде:
  - **Сурет 4. Деректер базасының ER-моделі (Кестелер қатынасы).**
    *(Сызбада: `User`, `Order`, `OrderItem`, `Product` кестелерінің квадраттық құрылымы мен олардың арасындағы 1-to-N (бірден-көпке) байланыс стрелкалары).*

  - **Сурет 5. API Gateway маршрутизация логикасының блок-схемасы.**
    *(Сызбада: Келген URL-ге байланысты трафиктің `/api/products` пен `/api/orders` микросервистеріне қалай бөлінетіні).*

- **2.3 Жүйе компоненттерінің өзара әрекеттесуін ұйымдастыру және іске асыру** бөлімінде:
  - **Сурет 6. Аналитика мен Excel есеп қалыптастыру алгоритмі.**
    *(Сызбада: Бэкенд дерекқордан мәліметті сұратып (Prisma), одан Buffer арқылы қалай `.xlsx` файлын жасап клиентке жүктеп беретінін көрсететін алгоритм).*

  - **Сурет 7. Telegram Bot хабарландыру алгоритмінің блок-схемасы.**
    *(Сызбада: "Жаңа тапсырыс келуі" -> "Дерекқорға жазылуы" -> "Telegram API-ге сұраныс жіберу" -> "Әкімші чатына хат баруы" процесі).*

  - **Сурет 8. CI/CD үздіксіз жеткізу және Docker контейнерлерді орналастыру схемасы.**
    *(Сызбада: "Бағдарламашы" -> "GitHub (push)" -> "Render бұлтты платформасы" -> "Auto-build" -> "Nginx арқылы клиентке ұсынылуы").*

---

## 3 Тарау. Қосымшаны жүзеге асыру және тестілеу бөлімі
Бұл бөлімде ДАЙЫН ЖҰМЫС ІСТЕП ТҰРҒАН САЙТТЫҢ және редактордың нақты СКРИНШОТТАРЫ орналасады.

- **3.1 Бағдарламалық жүйені жүзеге асыру** бөлімінде:
  - **Сурет 9. Монорепозиторий негізіндегі жобаның VS Code ішіндегі архитектуралық папкалар құрылымы.** 
    *(VS Code редактордың сол жағындағы `frontend/`, `api-gateway/`, `orders-service/` папкалары ашылып тұрған экранның скриншоты).*

  - **Сурет 10. Ағаш өңдеу цехы интернет-дүкенінің басты веб-интерфейсі (HomePage).** *(Сапалы үлкен скриншот).*
  - **Сурет 11. Электрондық каталог және тауар карточкасының дизайны.** *(Гридпен орналасқан тауарлар скриншоты).*
  - **Сурет 12. Split-screen архитектурасындағы авторизация (Login) беті.** *(Кіру/Тіркелу формасының скриншоты).*
  - **Сурет 13. Әкімшілік басқару панелі (Admin Dashboard) және аналитикалық график.** *(Dashboard метрикалары мен Recharts графикасы бар скриншот).*
  - **Сурет 14. Бағдарламаның сервері арқылы генерацияланған Excel (.xlsx) баяндама файлы.** *(Дайын ексель бағдарламасында ашылып тұрған тапсырыстар кестесінің скриншоты).*

- **3.2 Жүйені тестілеу нәтижелерін талдау** бөлімінде:
  - **Сурет 15. Zod валидация жүйесінің қате форма толтыру кезіндегі ескертулері (Alerts).** *(Мысалы, форманың ішіне қате мәлімет жазған кезде қыпылдап шығып тұратын қызыл қателердің скриншоты).*
  - **Сурет 16. Postman бағдарламасы арқылы API сұранысын тексеру нәтижесі.** *(Postman бағдарламасының ішінде "200 OK" және қайтып келген JSON жауабы (response) бар скриншот).*

  - **Сурет 17. LocalStorage ішіндегі қауіпсіз JWT-токеннің бейнеленуі.** *(Браузерді F12 арқылы ашып, DevTools ішіндегі Application -> LocalStorage бөлімінде токен жатқанын дәлелдейтін скриншот).*

- **3.3 Жүйенің жұмыс тиімділігін арттыру және бұлттық ортаға орналастыру** бөлімінде:
  - **Сурет 18. Render бұлтты платформасында жобаның сәтті компиляциялану (Continuous Deployment) процесі.** *(Render.com басқару панеліндегі сіздің сайтыңыздың жасыл "Live" немесе "Deployed" деген статусы бар бетінің скриншоты).*
